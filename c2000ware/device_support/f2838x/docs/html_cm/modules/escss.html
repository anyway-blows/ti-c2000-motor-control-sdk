ESCSS Module¶

The EtherCAT Sub-System (escss) API provides a set of functions for configuring and using the Sub System module. These contain EtherCAT Sub-System and Sub-System Control Registers. The Control registers configure GPIN and GPOUT Pad Select, Phy and Clock Source Select, Sync and Latch Signal Selects.

The following describes important details and recommendations when using the ESCSS API.

Once system control enables the ESCSS module, ESCSS_initMemory() needs to be called to put the controller in the init state, initialize the memory doesn’t get initialized. Next, use ESCSS_enableDebugAccess() which configures the debug accesses to be allowed to go through. The API ESCSS_enableGPIPipelinedRegCaptureOnEvent() enables the connection of GPIN to EtherCATSS through pipelined register as against the direct from IO. The ESCSS_setGPINData() sets the GPIN Data Register Value.

The code for this module is contained in driverlib_cm/escss.c, with driverlib_cm/escss.h containing the API declarations for use by applications.
